Установка плагина

1
Скачайте файл плагина

Нажмите кнопку ниже, чтобы скачать ChannelAlertPro.plugin.js

⬇ Скачать ChannelAlertPro.plugin.js
2
Поместите файл в папку плагинов

Выберите свою ОС:

🪟
Windows
%APPDATA%\BetterDiscord\plugins\
🍎
macOS
~/Library/Application Support/BetterDiscord/plugins/
🐧
Linux
~/.config/BetterDiscord/plugins/
3
Активируйте в BetterDiscord

Discord → ⚙️ Настройки → BetterDiscord → Plugins → Найдите ChannelAlertPro → переключите тоггл

4
Включите режим разработчика в Discord

⚙️ Настройки Discord → Дополнительно → Режим разработчика = ON
Это позволит копировать ID каналов: ПКМ на канале → Копировать ID

⚠️
Требования
  • • Десктопный Discord (Windows / macOS / Linux)
  • • BetterDiscord установлен: betterdiscord.app
  • • Не работает в браузерной версии Discord

Как создать правило

①
Откройте настройки плагина

⚙️ Настройки → BetterDiscord → Plugins → ⚙ у ChannelAlertPro

②
Нажмите «+ Добавить правило»

Новое правило появится внизу. Кликните на него — оно раскроется.

③
Заполните поля
Название

Любое имя для правила, например «Тикеты поддержки»

ID Канала

ПКМ на нужном канале в Discord → Копировать ID
(нужен режим разработчика в настройках Discord)

Текст-триггер

Слово или фраза. Например: [Тикет]: или @здесь
Плагин срабатывает, если сообщение содержит этот текст.

④
Выберите звук и нажмите «▶ Тест звука»

8 встроенных вариантов или загрузите свой MP3/WAV/OGG файл.

⑤
Включите тоггл правила

Зелёная точка = правило активно. Настройки сохраняются автоматически.

🎯
Несколько каналов

Создайте отдельное правило для каждого канала. Каждое правило работает независимо со своим триггером и звуком.

🔇
Быстрое отключение

Переключайте тоггл в заголовке правила — временно выключайте отдельные правила без их удаления.

🔤
Регистр триггера

По умолчанию регистр игнорируется. Включите «Учитывать регистр» если нужно точное совпадение.

💾
Автосохранение

Все настройки сохраняются автоматически через BdApi.Data. Правила не теряются при перезапуске.

8 встроенных звуков

Нажмите кнопку, чтобы прослушать звук прямо здесь. Все звуки генерируются через Web Audio API — файлы не нужны.

Загрузка своего звука

Поддерживаемые форматы: MP3, WAV, OGG, FLAC и любые другие, поддерживаемые браузером.

Тест загрузки звука

Выберите аудиофайл — он воспроизведётся прямо в браузере. В плагине аналогичный механизм.

Как это работает в плагине:
  1. 1. Файл читается через FileReader.readAsDataURL()
  2. 2. Base64 данные сохраняются в BdApi.Data (внутри плагина)
  3. 3. При срабатывании — декодируется и воспроизводится через AudioContext
  4. 4. ⚠️ Большие файлы увеличивают размер сохранённых данных. Рекомендуется до 500 КБ.

📡 Поиск FluxDispatcher (многоуровневый)
function findDispatcher() {
  const tries = [
    () => BdApi.Webpack.getByKeys('_actionHandlers', '_subscriptions'),
    () => BdApi.Webpack.getByKeys('actionLogger'),
    () => BdApi.Webpack.getByKeys('_dispatcher')?._dispatcher,
    () => BdApi.Webpack.getModule(m =>
      m && typeof m.subscribe === 'function'
      && m._subscriptions
    ),
  ];
  for (const fn of tries) {
    try {
      const r = fn();
      if (r && typeof r.subscribe === 'function') return r;
    } catch (_) {}
  }
  return null;
}
🎯 Обработчик сообщений
this._handler = (event) => {
  const msg = event?.message;
  if (!msg?.channel_id || !msg?.content) return;

  const rules = this.rules;
  for (const rule of rules) {
    if (!rule.enabled) continue;
    if (msg.channel_id !== rule.channelId) continue;

    const content = rule.matchCase
      ? msg.content
      : msg.content.toLowerCase();
    const trigger = rule.matchCase
      ? rule.trigger
      : rule.trigger.toLowerCase();

    if (content.includes(trigger)) {
      playSound(rule);  // звук
      if (rule.showToast)
        BdApi.UI.showToast(`🔔 ${rule.name}: ...`);
    }
  }
};

dispatcher.subscribe('MESSAGE_CREATE', this._handler);
🔊 Воспроизведение звука (с поддержкой кастомных)
function playSound(rule) {
  const ctx = new AudioContext();

  if (rule.customB64) {
    // Кастомный звук из Base64
    const bytes = base64ToBytes(rule.customB64);
    ctx.decodeAudioData(bytes.buffer, decoded => {
      const src  = ctx.createBufferSource();
      const gain = ctx.createGain();
      gain.gain.value = rule.volume;
      src.buffer = decoded;
      src.connect(gain).connect(ctx.destination);
      src.start();
    });
    return;
  }

// Загрузка правил
get rules() {
  return BdApi.Data.load(this.meta.name, 'rules') || [];
}

// Сохранение правил
saveRules(rules) {
  BdApi.Data.save(this.meta.name, 'rules', rules);
}
